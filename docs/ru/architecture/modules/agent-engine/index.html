Agent Engine

analytics · autonomous · configurable

Единый модуль автономных агентов поверх Knowledge Store. Сотрудник заводит личного агента — промт, расписание, инструменты к знаниям — и тот работает сам, под правами владельца. Библиотека аналитиков и командные агенты — следующий горизонт.

Архитектура

t0 Триггер
владелец инициирует прогон или срабатывает расписание
Агент = владелец перед KS · ACL целиком, pre-filter на стороне KS
t0 Сборка промта · 3 слоя
1 платформенныйsafety + организация 2 промт агентазадаёт владелец 3 инженерныйличность + tools
платформу промтом агента не перебить
t1 Гейт старта
enabled ∧ ¬admin_paused ∧ бюджет ∧ model_id≠NULL → старт
enabled · замок владельца admin_paused · липкий замок, админ
t2 Агентный цикл · ход ведёт модель
думает ▸ зовёт tool ▸ наблюдает ↻ снова
повторяется участком, не разовое событие
крутится до ответа или до потолка итераций · agent_iteration_cap
ядро · 3 read-only примитива KS: search graph sql + выбор владельца из каталога
Knowledge Store · read-only инструменты тянут из знаний — в KS агент не пишет
Недельный бюджет токен-потолок на пользователя · расходуется по ходу цикла, живёт дольше одного прогона
t3 Остановка
ответ получен или достигнут потолок итераций
t4 Запись журнала
Журнал прогонов
agent_runs.output · tokens_used
Не workflow-конструктор — модель комбинирует инструменты по намерению, не граф узлов перед промтом агента — платформенный prompt: safety и правила организации, единые с чатом
