Notifications

cross-cutting · dispatcher · in-app · email · webhook

Сквозной слой уведомлений: один диспетчер в core собирает события всех модулей, разводит по двум моделям адресации и каналам, журналирует доставку. Своего экрана у модуля нет — ленту и настройку рендерят surface'ы (указатель ниже).

Макеты Surface'ы

Своего фронтенда у модуля нет: ленту читает каждый surface своим экраном — → Web App у сотрудника, → Admin Panel у администратора; настройку каналов и маршрутов держит → экран уведомлений Admin Panel.
