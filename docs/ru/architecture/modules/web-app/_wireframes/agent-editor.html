Дайджест релизов модель снята
Модель
Выбрать модель
  1. 1 Выключатель владельца — «включён» оставляет агента на расписании и ручном запуске, «выключен» останавливает целиком. Сводный статус — в заголовке.
  2. 2 «Запустить сейчас» — внеочередной прогон с текущими настройками; чтит выключатель, замок админа и бюджет. Один активный прогон на агента: поверх идущего — пропуск. Agent Engine.
  3. 3 Промт владельца — задача, формат вывода, ограничения. Ложится поверх платформенного слоя (safety + правила организации), перебить его нельзя. Agent Engine.
  4. 4 Ядро из трёх внутренних инструментов — поиск, граф, SQL — фиксировано и всегда под личностью владельца: что закрыто сотруднику, закрыто и агенту, запись в граф недоступна. Agent Engine.
  5. 5 Расписание — режим плюс параметр под него (превью режимов выше): «вручную» — без параметра, интервал — пресет «каждые N часов», календарно — слот «ежедневно / еженедельно в HH:MM», резолвится против пояса владельца. Сырой cron и событийные триггеры — v2. Agent Engine.
  6. 6 Журнал прогонов — время, повод, исход, длительность и расход токенов; строка раскрывается в полный вывод агента.
  7. 7 Исход прогона: succeeded · failed (ошибка / потолок итераций) · skipped (лимит недели / уже идёт). Машина состояний — Agent Engine.
  8. 8 Замок администратора липкий: пока он стоит, выключатель и запуск недоступны владельцу. Agent Engine.
  9. 9 Модель снята администратором из списка — агент остановлен, пока владелец не выберет другую из актуального списка. Прежний выбор на дефолт не откатывается. Agent Engine.
  10. 10 Внешние инструменты — выбор из разрешённых администратором read-only пресетов (web_search, fetch_url); снятие галочки не блокирует прогон, агент продолжает на ядре KS. Подключение MCP-серверов — v2. Admin Panel.
