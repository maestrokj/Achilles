  1. 1 Колокольчик с unread-счётчиком — число непрочитанных in-app у текущего пользователя (доставки с read_at IS NULL). Счётчик обновляется мгновенно push-каналом (SSE), опрос — фоновый добор. Канал in-app залочён включённым, поэтому колокольчик есть у каждого — лента = notifications ⋈ свои deliveries.
  2. 2 Панель — хвост последних событий, по клику открывается из колокольчика. «Прочитать все» проставляет read_at всем доставкам пользователя разом; шестерёнка ведёт в личные настройки уведомлений (тумблеры in-app / email по типу).
  3. 3 Тип слева (Agent · Account) — категория личного события; severity-чип — серьёзность. Источник кликабелен: диплинк из source_ref ведёт на сущность — прогон, агента, профиль.
  4. 4 Единственный actionable-случай у сотрудника — снятая модель: кнопка ведёт в редактор агента, выбрать модель из доступных. Прочие личные события — информирующие, с диплинком «куда смотреть».
  5. 5 «Показать все» ведёт на полный экран — история, фильтры, отметка прочтения по одному. В панели — только верхушка.
  6. Лента личная: сюда падают только targeted-события этого пользователя (его агенты, его учётка). Организационные алерты (sync · security · budget) — у администраторов в Admin Panel.

  1. 6 Фасеты Тип · Severity · Непрочитанные · Период — поведение единое для всех списков. «Непрочитанные» — частый дефолт; снять, чтобы увидеть историю. Тип здесь — два личных: Agent · Account.
  2. 7 Severity-чип = notifications.severity (warning · info), цвет из каталога событий. Тип слева — категория (Agent · Account).
  3. 8 Источник кликабелен — диплинк на сущность из source_ref: прогон, агент, профиль. Уводит туда, где разобраться, прямо из ленты.
  4. 9 «Отметить прочитанным» пишет read_at в собственную доставку — у каждого своя лента и свой read-state. Само событие — append-only, его не правят и не удаляют.
  5. 10 Прочитанное гаснет (приглушённая карточка, без акцентной полосы), но остаётся в истории; «Отметить непрочитанным» возвращает в счётчик. Группировка по дню — Сегодня · Ранее.
