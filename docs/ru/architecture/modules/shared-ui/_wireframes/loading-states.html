Загрузка и ожидание

shared ui · макеты · общий компонент

Словарь механизмов ожидания внутри живого экрана: механизм под форму ожидания, а не под тип экрана. Полный обвал — отдельный паттерн, системные заглушки.

Какой механизм когда

  • Форма контента известна, данных ещё нет → skeleton
  • Контент уже на экране, меняется срез → keep-previous
  • Область крошечная или форма неизвестна → спиннер
  • Меняется экран целиком → полоса сверху
  • Запрос упал, каркас жив → инлайн-ошибка с повтором

Skeleton — холодный старт → Web App

Каркас сразу, тело — плейсхолдеры в форме будущего контента: при подмене на данные нет скачка вёрстки.

Полоса — смена экрана целиком

Тонкая полоса под шапкой, пока грузится следующий маршрут. Живёт на уровне оболочки (layout), не на контентных экранах.

Инлайн-ошибка — повтор

Блок с причиной и «Повторить» в области данных, каркас экрана жив. Полный обвал — системная заглушка; невидимый фоновый сбой — тост.

Пользователи
Не удалось загрузить список Сервер не ответил. Проверьте соединение и повторите.

Правила

  • Порог ~200 мс — индикатор не показываем, если ответ успел прийти быстрее: иначе экран мерцал бы вспышкой лоадера.
  • Каркас не дёргается — заголовок, фильтры, пагинация рендерятся сразу; меняется только область данных.
  • Доступность — зона загрузки несёт aria-busy, инлайн-ошибка — role="alert", чтобы скринридер сообщал о смене состояния.
