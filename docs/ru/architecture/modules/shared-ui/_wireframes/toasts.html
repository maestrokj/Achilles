Toasts

shared ui · макеты · общий компонент

Короткое всплывающее сообщение об исходе действия — для того, что пользователь иначе не увидит.

Когда показывать

  • ✓ Ошибка / сбой — системная или фоновая. Ошибки формы — inline у поля.
  • ✓ Фоновый / отложенный исход — синхронизация запущена, письмо в очереди.
  • ✓ Изменение вне видимости — элемент, с которого ушли; массовое действие.
  • ✓ Значимое действие без видимого следа — отзыв доступа, отправка приглашения.
  • ✕ Результат виден на месте — тост дублирует; подтверждение inline.
  • ✕ Рутинное, мгновенно обратимое — тумблер, разворот блока: фидбэк не нужен.

  1. success — значимое действие прошло: отправлено, создано, отозвано.
  2. error — действие не прошло; причина + опциональная ссылка к исправлению.
  3. info — нейтральный статус: фоновая задача запущена.
  4. warn — выполнено с оговоркой.

  • Позиция — нижний правый угол, поверх контента, без сдвига layout.
  • Авто-скрытие — ~5 секунд; наведение/фокус паузят таймер, × закрывает вручную.
  • Стек — столбцом, свежий снизу; сверх лимита старые сворачиваются.
  • Доступность — role="status" или role="alert" (error), aria-live для скринридеров.
