Defaults организации задаёт Owner на экране уведомлений; здесь — личное сужение. У личных типов email по умолчанию выключен — opt-in.
  1. 1 Email — read-only, логин-идентификатор. Меняет администратор в Admin Panel.
  2. 2 Роль — read-only, свою роль не меняют. Назначает Owner / Admin в Admin Panel.
  3. 3 Язык и регион — личный override дефолтов организации (timezone, язык UI, формат дат). Влияет только на отображение. Механика — Язык и регион.
  4. 4 MFA v2 — обязателен для Owner / Admin, опционален для Member. Настройка — Двухфакторная аутентификация.
  5. 5 Активные сессии — список устройств с отзывом по одному. Экран — управление сессиями.
  6. 6 «Выйти на всех устройствах» завершает все сессии; обычный «Выйти» — только текущую. Механика — Завершение сессии.
  7. 7 Правила пароля — Парольная политика. Новый пароль обязан отличаться от текущего.
  8. 8 Индикатор силы — оценка надёжности по мере ввода.
  9. 9 Смена пароля требует ввода текущего. Механика — Смена пароля. v2: текущий пароль заменяет sudo-mode через re-auth-modal.
  10. 10 API-ключи — пользователь сам выпускает и отзывает свои ключи. «Создать ключ» открывает диалог создания. Механика — API-ключи.
  11. 11 На каждый тип два независимых тумблера, in-app и email; выключить можно любой или оба, привязки и замков нет. Организационные типы (Sync · Security · Budget · System · Discovery) видны только Owner / Admin — лишь они адресаты этих алертов. Общие webhooks команды (Slack / Ops) и defaults организации — Уведомления в Admin Panel.
  12. 12 Личные типы (Agent — прогоны его агентов; Account — роль, временный пароль) приходят каждому пользователю, включая member — это targeted-события на самого человека. Email у них по умолчанию выключен (opt-in); in-app виден сразу.
