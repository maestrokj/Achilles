Восстановление пароля

auth / security · фичи

Сброс пароля по email-ссылке — двумя путями: пользователь запрашивает ссылку самостоятельно («забыл пароль» на экране входа), либо администратор отправляет её из карточки пользователя. Дальше путь общий: переход по ссылке из письма, установка нового пароля, завершение всех прежних сессий.

Зависимости

Оба сценария требуют настроенного SMTP — секция в Admin Panel (настройки платформы). Пока SMTP не настроен, действует фолбэк: самостоятельного сброса нет, а администратор выдаёт временный пароль — модал в Admin Panel, CLI или Owner.
Механика — в workzone: сброс «забыл пароль» · сброс администратором · парольная политика · рубеж защиты от перебора.
