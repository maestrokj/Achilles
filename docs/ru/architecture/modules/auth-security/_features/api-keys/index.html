API-ключи

auth / security · фичи

Выпуск API-ключа — машинного (headless) доступа к платформе: им аутентифицируется Public API, в следующей версии — MCP v2. Особенность фичи не в длине пути, а в его форме: один диалог создания, в который ведут три двери. Пользователь выпускает ключ на себя из своего профиля (самовыпуск, без выбора пользователя); Owner / Admin выдаёт ключ на сотрудника из общего списка ключей или из карточки конкретного сотрудника (выдача, с поиском-подсказками). Все двери сходятся в одном диалоге создания, который существует единственной копией. Самовыпуск — основной путь, выдача администратором — вспомогательный; ключ никогда не шире прав своего владельца.
