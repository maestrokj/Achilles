Принудительная смена пароля

auth / security · фичи

Пользователь, которому администратор сбросил пароль без настроенного SMTP, получает временный пароль. При первом входе по нему система не пускает в приложение: уводит на экран смены и держит там, пока не задан постоянный пароль. Только после этого открывается рабочая область по роли.
