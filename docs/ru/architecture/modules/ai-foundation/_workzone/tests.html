Тест-кейсы

ai-foundation · workzone
Принятые решения
Реестр в core, читатели только читают Строки заводят Admin (провайдеры / модели) и рантайм (расход); потребители — Harvester, Query Engine, Agent Engine — из реестра лишь читают, строк не создают. Констрейнты бьём через ORM прямо в тест-БД: testcontainers PostgreSQL, реальная схема из миграции, per-test rollback.
RESTRICT — источник истины целостности назначений Занятую модель нельзя удалить или переназначить, пока на неё ссылаются model_assignments / chat_models / agent_models. Замок держит БД (FK RESTRICT), не только проверка в коде — тест обходит сервисный слой и бьёт прямо в базу.
Границы охвата — только БД Здесь контрольный слой: констрейнты, каскады, upsert-агрегат, засев и обратимость. Коннективити-чек провайдера и discovery моделей — логика модуля, её покрывает отдельная итерация; тут не трогаем.
Стек и инфраструктура
Имеется done pytest, pytest-asyncio
Тест-БД testcontainers (PostgreSQL) — реальная схема, per-test rollback; констрейнты проверяются в живой базе, не в моках
Засев проверяется прогоном самой миграции — Platform-провайдер, встроенные модели, tool-пресеты; не хардкод-фикстурой
Маркеры @pytest.mark.integration — весь слой требует БД; приоритет (p0 / p1) ортогонально типу
