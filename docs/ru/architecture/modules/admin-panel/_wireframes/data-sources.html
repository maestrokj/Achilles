Идёт платформенная доводка графа (мёрж сущностей) — запуск встанет в очередь и стартует по её завершении. 12
  1. 1 Тип и набор полей задаёт манифест коннектора. Base URL — адрес инстанса источника.
  2. 2 Две независимые оси: Учётная запись (сервисная / личная) и Способ (статический токен · OAuth v2). Комбинации — Harvester.
  3. 3 Учётные данные под маской — полное значение видно только при создании. Хранятся зашифрованными крипто-ядром Auth.
  4. 4 «Проверить» гоняет пробу из трёх шагов — связь, креды, права — и даёт вердикт по каждому, чтобы починить ровно оборвавшееся звено. Та же проба идёт по расписанию: провал уводит источник в error и поднимает уведомление. Health-check — Harvester.
  5. 5 Scope — режим, не ручной перечень: «Всё» берёт источник целиком и подхватывает новое, «Только выбранное» — отбор из каталога. Режимы — Harvester.
  6. 6 Базовые правила встроены и не настраиваются. Переключатели «сверх правил» — релевантные типу источника (у Jira вложения и комментарии): вернуть отсеянное или добрать необязательный контент. Набор объявляет манифест коннектора.
  7. 7 Webhooks — источник шлёт событие, коннектор тянет дельту вне опроса; канал один на источник. Endpoint и секрет админ вставляет вручную (авто-провижн — v2). Сменить секрет — за подтверждением, значение видно один раз; ротация с grace-окном идёт без простоя. Приём входящих держит security.
  8. 8 По умолчанию расписание наследует глобальный дефолт из хаба; «Своё расписание» раскрывает поля и пишет override на источник. Инкремент — интервал без окна; у reconciliation override — окно запуска (частота, день, час) в таймзоне организации. Строка под полями разворачивает выбор в живую фразу.
  9. 9 Авторитет источника — базовый вес доверия в ранжировании: при равной свежести регламент перевесит реплику в чате. Стартовое значение по типу даёт манифест, здесь админ его переопределяет. Дальше его берёт за базу Curation Pass — это ранжирующий вес, не фильтр доступа.
  10. 10 История прогонов свёрнута по умолчанию — вторична рядом с конфигурацией. Один прогон = одна запись SyncRun: состояние, прогресс «N из M», ошибки. Сортировка — клик по заголовку колонки. Тип Partial re-sync — самовосстановление инкремента; череда таких подряд сигналит, что webhook источника барахлит.
  11. 11 Пауза глушит расписание; Отключить — обратимая деактивация с сохранением конфига и данных; Удалить за type-to-confirm предлагает выбор «только конфигурация» / «конфигурация + данные» (необратимо). Состояния — Harvester.
  12. 12 Запуск может встать в queued из-за платформенной доводки графа: на её разрушительных шагах синхронизация ждёт и стартует по завершении — не падает. Координация полос — Knowledge Store.
