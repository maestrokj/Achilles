Активные сессии завершатся; пользователь войдёт заново по новому адресу.

  1. 1 Смена email — админ задаёт новый адрес сотрудника. Гейт — модал. Механика — Auth & Security.
  2. 2 Смена роли — назначить Owner может только Owner, Admin управляет только Member; единственного Owner понизить нельзя. Auth & Security.
  3. 3 Identity mapping — ручная привязка к source-аккаунтам там, где авто-матч не справился. Модель — Knowledge Store.
  4. 4 Обзор доступов v2 — что пользователь видит по каждому источнику и какие платформенные ограничения применяются. Auth & Security.
  5. 5 Персональные permissions v2 — точечное расширение или урезание прав поверх роли. Auth & Security.
  6. 6 Сессии — админ видит число активных сессий; деталей (устройство, IP) нет. Единственное действие — «завершить все». Auth & Security.
  7. 7 Действия с пользователем критичны — с v2 перед выполнением админ подтверждает пароль через модал re-auth.
  8. 8 Сброс пароля — админ отправляет на email пользователя ссылку для установки нового пароля; без SMTP — фолбэк временным паролем. Поток — Восстановление пароля.
  9. 9 Деактивация и удаление — два разных действия. Деактивация закрывает вход обратимо; удаление необратимо стирает учётку и доступно только Owner. Auth & Security.
  10. 10 API-ключи — ключи этого пользователя; «Создать ключ» открывает общий диалог создания. Вспомогательный путь — полный список по компании в API Keys.
