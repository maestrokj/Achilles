  1. 1 Колокольчик с unread-счётчиком — число непрочитанных in-app у текущего админа (доставки с read_at IS NULL). Счётчик обновляется мгновенно push-каналом (SSE), опрос — фоновый добор. Канал in-app залочён включённым, поэтому колокольчик есть всегда — лента = notifications ⋈ deliveries.
  2. 2 Панель — хвост последних событий, по клику открывается из колокольчика. «Прочитать все» проставляет read_at всем доставкам админа разом; шестерёнка ведёт в настройки каналов и маршрутов (правит только Owner).
  3. 3 Дедуп — серия однотипных сбоев схлопнута в одно уведомление со счётчиком ×N, не N строк. Ключ и окно — тротлинг диспетчера.
  4. 4 Discovery — единственный actionable-тип: «В scope» прямо из ленты заводит контейнер в охват источника (payload в notifications.meta). Событие общее для всех Owner / Admin, а действие глобальное, поэтому оно идемпотентно: источник истины по охвату — allow-list Harvester; контейнер уже в scope → повтор ничего не делает, а кнопка «В scope» гаснет у всех. Само уведомление состояние действия не хранит — журнал не мутируется ради этого. Живёт только в режиме scope «Только выбранное»; в режиме «Всё» объект подхватывается сам — тогда это просто инфо.
  5. 5 «Показать все» ведёт на полный экран — там история, фильтры и отметка прочтения по одному. В панели — только верхушка.
  6. Это лента Admin Panel — для Owner / Admin, адресатов алертов организации. У сотрудника-member своя лента личных уведомлений в Web App (прогоны агента, смена роли) — тот же диспетчер, своя витрина.

  1. 6 Фасеты Тип · Severity · Непрочитанные · Период — поведение единое для всех списков. «Непрочитанные» — частый дефолт; снять, чтобы увидеть историю.
  2. 7 Severity-чип = notifications.severity (critical · warning · info), цвет из каталога событий. Тип слева — категория маршрутизации (Sync · Security · Budget · System · Discovery).
  3. 8 Источник кликабелен — диплинк на сущность из source_ref: прогон, источник, провайдер, бюджет. Уводит туда, где чинят, прямо из ленты.
  4. 9 «Отметить прочитанным» пишет read_at в собственную доставку админа — у каждого своя лента и свой read-state, чужие не трогаются. Само событие — append-only, его не правят и не удаляют.
  5. 10 Прочитанное гаснет (приглушённая карточка, без акцентной полосы), но остаётся в истории; «Отметить непрочитанным» возвращает в счётчик. Группировка по дню — Сегодня · Ранее.
  6. Что попадает в ленту — личное: любой тип и канал, включая in-app у security и sync, отключается у себя в профиле (notification_prefs), личных замков нет. Организационный замок этих двух держит in-app включённым лишь на уровне организации — централизованно не погасить, но личного выбора он не ограничивает.
