1 Разрешение локали timezone · язык UI
Цепочка приоритета
При показе дат, времени, языка интерфейса и формата фронтенд берёт первое заданное значение: личный override пользователя → дефолт организации → locale браузера. Локаль влияет только на отображение, не на сами данные.
Дефолт организации → Platform Settings
Owner / Admin задаёт timezone, locale, date_format в platform_settings (singleton). Применяется ко всем, кто не переопределил, — и служит точкой отсчёта для расписаний sync, агентов и digest-ов.
Личный override → Профиль
Пользователь переопределяет под себя в профиле. Хранится в per-user полях timezone / locale / date_format на users (nullable; NULL = наследовать дефолт организации). Незаданное поле прозрачно падает на уровень организации — переопределить можно и одну строку, не все три.
