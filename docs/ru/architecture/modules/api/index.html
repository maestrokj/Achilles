Поверхности API

Внутренний /api/v1 — наши оболочки (Web App, Admin Panel).
Внешний /public/v1 — курируемый срез того же движка (Public API).
Конкретный контракт — в tests.html владельца.

Admin Panel и Web App — фасады: своих доменных ресурсов не держат, сводят контракты владельцев под ролью. Строка Admin — про конфиг-поверхность /admin/*, не про владение.

Общие правила
  • Версия в пути — /api/v1 · /public/v1; ломающее → новая версия
  • Личность — JWT или ключ ach_… → user_id → роль → ACL
  • Ошибка — код, не 500 — 401 · 403 · 404 · 409 · 422 · 429, единая форма
  • Списки — единый контракт пагинации/фильтра/сортировки (list-controls)
  • Диалог — поток SSE: токены инкрементами, цитаты в хвосте
  • Живые борды — сервер шлёт сигнал, клиент рефетчит по REST (Живые обновления)
  • Идемпотентность — очередное по job_id: повтор не задваивает
  • Лимит частоты — окно → 429 (Cache & Workers)
  • Время — UTC ISO 8601 (…Z); пояс применяет фронт
Workzone Проектирование

Собственная механика слоя: платформенный push-канал, который держит живые борды свежими, и conformance-набор, исполняющий правила выше — форма ошибки, UTC, пагинация и лимит частоты проверяются параметрически по всем роутерам, а не копией в каждом модуле.
