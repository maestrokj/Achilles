Cache & Workers

кэш · очереди · планировщик · воркеры · rate-limit

Эфемерная инфраструктура платформы поверх Redis: ускоряет (кэш частых чтений) и связывает (асинхронная передача задач между модулями). Своих доменных знаний не хранит — это общий слой, который остальные модули потребляют через тонкий клиент. Два лица: синхронное эфемерное состояние на пути запроса (кэш, rate-limit, blacklist) и фоновая исполнительная среда вне запроса (очередь, планировщик, воркеры, жизненный цикл задачи).

Архитектура

Два лица модуля бок о бок: синхронное на пути запроса и фоновое вне его, каждое со своим потоком сверху вниз. Оба держатся на общем фундаменте из трёх хранилищ; статический принцип «что где живёт» — на водоразделе Redis ⟂ Postgres.
