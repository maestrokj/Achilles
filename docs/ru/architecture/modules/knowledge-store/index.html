Knowledge Store

персистентное хранение знаний

Multi-model хранилище: одна сущность знаний живёт тремя проекциями в одном Postgres — реляционное тело, векторы фрагментов, граф связей. Это retrieval-слой платформы: Harvester сюда пишет, Query Engine отсюда берёт контекст для RAG. Отличительная черта — слияние четырёх парадигм поиска под единым ACL одним SQL-запросом.
