Тест-кейсы

email · workzone
Принятые решения
SMTP мокается Реальная отправка не происходит. Проверяем, что письмо собрано верно (адрес, тема, ссылка, язык, multipart/alternative) и что send вызван. Тот же подход уже применён в auth-тестах.
SAQ-задача — вызов обработчика напрямую Без поднятия воркера: тестируем логику задачи (compose + send, ретраи, разметка «не доставлено»), не инфраструктуру очереди.
Стек и инфраструктура
Имеется done pytest, pytest-asyncio, pytest-cov, httpx
Добавить testcontainers (PostgreSQL), мок SMTP (подмена aiosmtplib), time-machine (TTL ссылок и таймауты)
Тест-БД singleton smtp_settings — реальная строка в тест-БД, per-test rollback

test_queue_task.py
Очередь SAQ — обработчик задачи напрямую
IntegrationP0 → Inline или очередь
Кейсы

IntegrationP1 → Ошибки доставки
Кейсы
4xx / сеть → ретрайвременный сбой → повтор с экспоненциальным backoff

ретраи исчерпанызадача в «не доставлено», статус виден потребителю

backoff растётзадержка между попытками увеличивается, ≤ предел
