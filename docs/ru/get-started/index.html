Achilles

Get Started

Два консольных шага — развернуть и создать владельца. Дальше платформа ведёт сама: на дашборде вас ждёт панель настройки со ссылками на нужные экраны и галочками выполненного.

1 Развёртывание Поднять платформу на своём сервере — Docker Compose и три команды. 2 Первый вход Создать владельца платформы и открыть панель администратора. 3 Панель настройки Шесть шагов на дашборде: AI-модели, источники, почта, мессенджеры — прогресс панель отмечает сама. 4 Приглашение команды Позвать сотрудников по email, раздать роли, привязать мессенджеры. 5 Тонкая настройкаопционально Организация, лимиты расходов, инструменты AI, промпт организации, уведомления. 6 Кастомизацияопционально Свои поверхности через Public API, MCP, новые коннекторы и инструменты.
Перед началом

Что понадобится

Linux-сервер (VPS) с установленными Docker и Docker Compose — от 2 vCPU и 8 ГБ RAM (точнее — в шаге 1).
Домен, A-запись которого указывает на сервер.
API-ключ AI-провайдера — Anthropic, OpenAI, Google, агрегатор нейросетей (OpenRouter и подобные) или свой OpenAI-совместимый сервер, включая локальный Ollama.
Доступ к SMTP (почтовый сервер для исходящих писем) — нужен, чтобы приглашать сотрудников. Можно добавить позже.
Консоль · шаг 1

Развёртывание

Платформа поставляется как набор Docker-контейнеров: база данных, очереди, backend, frontend и reverse-proxy поднимаются одной командой.

Требования к серверу минимум · рекомендуется
AI-хаб (без базы знаний) 2 vCPU · 8 ГБ RAM · 40 ГБ SSD
С базой знаний и встроенным эмбеддером 8 vCPU · 16 ГБ RAM · 80 ГБ SSD
встроенный эмбеддер держит модель в памяти (пик ~6 ГБ); если эмбеддинги считает внешний провайдер — достаточно минимума. Запускаете эмбеддер на хосте с 8 ГБ? Добавьте swap (4 ГБ хватит) — пик загрузки кратковременный, но без swap сервис может уйти в OOM
1
Склонировать репозиторий и заполнить настройки
Файл .env — единственное место, которое нужно отредактировать перед запуском.
git clone https://github.com/maestrokj/Achilles.git cd Achilles cp .env.example .env
.env заполнить эти строки
POSTGRES_PASSWORD / REDIS_PASSWORD надёжные пароли
SECRET_KEY openssl rand -base64 32
CORS_ORIGINS ["https://ваш-домен.com"]
DOMAIN ваш-домен.com
PUBLIC_BASE_URL https://ваш-домен.com
VITE_API_URL https://ваш-домен.com/api
2
Запустить платформу
docker compose -f docker-compose.prod.yml up -d
Проверка: откройте http://<ваш-домен>/ — платформа встретит страницей статуса.
3
Выпустить TLS-сертификат
Certbot уже входит в поставку. После выпуска сертификата nginx сам переключится на HTTPS; продление — автоматическое.
docker compose -f docker-compose.prod.yml run --rm --entrypoint certbot certbot \ certonly --webroot -w /var/www/certbot -d <ваш-домен> -d www.<ваш-домен> docker compose -f docker-compose.prod.yml restart nginx

Консоль · шаг 2

Первый вход

У свежей платформы нет ни одного аккаунта. Первый — владелец (Owner) — вы создаёте сами; все остальные потом войдут по приглашениям. Сделать это можно двумя способами, и оба работают только пока база пуста.

A
В браузере — экран первого запуска
Просто откройте https://<ваш-домен>/. На пустой базе платформа сразу приводит вас на экран создания владельца — введите имя, почту и пароль, и вы попадёте в панель администратора уже под своей учётной записью.
B
В консоли — командой
Не хотите держать форму на публичном домене? Создайте владельца на сервере, затем войдите на https://<ваш-домен>/login.
docker compose -f docker-compose.prod.yml exec backend \ achilles init-owner --email you@company.com --password '…' --name 'Ваше имя'
Экран первого запуска открыт тому, кто первым дойдёт до домена, — поэтому создайте владельца сразу после развёртывания. Как только появляется хоть один аккаунт, и экран, и команда навсегда отключаются, а домен показывает обычный вход.
Роли в платформе: Владелец — всё, включая журнал аудита и настройки безопасности; Администратор — панель управления; Участник — чат, агенты и личный кабинет.
Дальше — в браузере

Панель настройки ведёт сама

После первого входа дашборд администратора встречает панелью настройки: шесть шагов, каждый — ссылка на нужный экран. Галочки платформа ставит сама по мере выполнения, а когда всё готово, панель исчезает. Разделы ниже проходят те же шесть шагов в том же порядке.

Панель · 1

Модели для ответов

Сердце платформы. Один провайдер и одна модель — минимум, с которым чат уже работает. Всё происходит на одном экране — https://<ваш-домен>/admin/ai-models (пункт меню «AI-модели»).

1
Добавить провайдера
«Добавить провайдера» → вставьте API-ключ → «Проверить». Подойдёт крупный провайдер (Anthropic, OpenAI, Google), агрегатор нейросетей или своя OpenAI-совместимая модель — для неё достаточно указать адрес сервера.
Новый провайдер Проверить
Провайдер Anthropic
API-ключ sk-ant-•••••••••••••
2
Включить модели
Включите нужные модели из каталога провайдера.
3
Назначить их чату
В блоке «Назначения» отметьте, какие из включённых моделей доступны сотрудникам в чате.
Чат работает. Откройте https://<ваш-домен>/chat и попробуйте — остальные шаги панели его не блокируют. Это рабочее место сотрудников: здесь они общаются с AI, а рядом заводят и настраивают своих агентов.

Панель · 2 для базы знаний

Модель эмбеддинга

Тот же экран «AI-модели», блок «Назначения». Модель эмбеддинга превращает документы в вектора для поиска — без неё база знаний не соберётся. В поставку входит встроенный эмбеддер, который работает на вашем сервере, — внешний провайдер не обязателен.

Выберите модель до первой синхронизации источников: смена позже означает переиндексацию всей базы.
Панель · 3 для базы знаний

Источники данных

Этот шаг превращает платформу из AI-хаба в память компании: Jira, Confluence и другие источники собираются в граф знаний, и ответы начинают опираться на ваши документы — со ссылками на первоисточник. Экран — https://<ваш-домен>/admin/harvester (пункт меню «Источники данных»).

1
Добавить источник
Кнопка «Добавить источник» открывает мастер: тип источника, доступы, проверка соединения, охват.
2
Запустить синхронизацию
Вручную или дождитесь расписания. Интервал инкрементальной синхронизации общий для всех источников; отдельному источнику можно задать своё расписание в его карточке.
3
Проверить, что люди сопоставились
После первой синхронизации платформа сама связывает аккаунты источника с пользователями по email. Тех, кого связать не удалось, видно на дашборде («непривязанные аккаунты») и на экране https://<ваш-домен>/admin/users → вкладка «Связь с источниками данных» — там же связь ставится вручную в пару кликов. Пока аккаунт не связан, сотрудник получает в ответах только общедоступные документы.
4
Присматривать за хранилищем
https://<ваш-домен>/admin/knowledge-store (пункт «Хранилище») — состояние графа (сущности, фрагменты, связи, векторы), расписание доводки, политика хранения и резервные копии во внешнее хранилище.
Права доступа источников переносятся в граф: сотрудник получает в ответах только те документы, которые ему доступны в первоисточнике.
Как это устроено: Harvester Knowledge Store

Модели для агентов

Агенты — автономные помощники, которых сотрудники заводят для регулярной работы на экране https://<ваш-домен>/agents. На экране «AI-модели», в том же блоке «Назначения», отметьте модели, доступные агентам, — из этого списка сотрудник выбирает модель для своего агента.

Панель · 6 опционально

Каналы общения

Сотрудники могут спрашивать платформу прямо из мессенджера — каждый тред становится отдельной беседой. Карточки живут на экране «Платформа», в блоке «Поверхности»; подключайте любую из трёх или все сразу.

Поверхности Платформа · что нужно каждой
Slack bot token · signing secret
Mattermost адрес сервера · bot token
Telegram bot token
у каждой карточки своя кнопка «Проверить» — платформа сразу проверяет доступы
Чтобы платформа узнавала, кто пишет, сотрудник привязывает свой аккаунт мессенджера в личном кабинете — см. «Приглашение команды» ниже.
Все шесть галочек — панель исчезает сама. Платформа настроена; осталось привести людей.

Почти готово

Приглашение команды

Аккаунты создаются только по приглашению — самостоятельной регистрации нет. Понадобится настроенная почта (шаг 5 панели); если SMTP не настроен, экран приглашений сам об этом напомнит.

1
Отправить приглашение
https://<ваш-домен>/admin/users (пункт «Пользователи») → вкладка «Приглашения» → email и роль. Ссылка в письме действует 48 часов.
Пригласить сотрудника Отправить
Email colleague@company.com
Роль Участник
2
Сотрудник принимает приглашение
По ссылке из письма он задаёт имя и пароль — и попадает в чат. Свои агенты, история бесед, лента уведомлений и личный кабинет уже ждут его.
3
Привязка мессенджера опционально
В личном кабинете (https://<ваш-домен>/account, раздел «Связанные аккаунты») сотрудник выбирает мессенджер — платформа выдаёт короткоживущий код, его нужно отправить боту в личные сообщения, и аккаунты связаны.
Тонкая настройка опционально

Сделать платформу своей

За пределами панели настройки — стоит заглянуть, когда платформа уже работает.

Организация
Название компании, логотип, часовой пояс, язык интерфейса и формат дат — дефолты, которые каждый сотрудник может переопределить. /admin/platform
Лимиты расходов
Недельный потолок токенов для агентов и месячное бюджетное оповещение; здесь же виден расход по сотрудникам и моделям. /admin/ai-usage
Инструменты AI
Веб-поиск и загрузка страниц для чата и агентов. /admin/ai-tools
Промпт организации
Тон, правила и контекст компании, которые AI учитывает в каждом ответе. /admin/ai-prompt
Уведомления
Какие события до кого доходят и по каким каналам — лента внутри платформы и почта. /admin/notifications
Расти дальше опционально

Кастомизация

Платформа расширяется в четырёх направлениях — каждое описано в архитектурной документации.

Свои поверхности — Public API
Внутренний портал, другой мессенджер, свой инструмент — любая поверхность подключается через открытый HTTP-API. Ключи доступа выпускаются на экране «API-ключи» (/admin/api-keys).
Внешние AI-ассистенты — MCP
Claude, IDE и другие MCP-клиенты подключаются к базе знаний компании как к инструменту — включается рубильником на экране «Платформа».
Свои коннекторы
Нестандартный источник данных подключается по контракту Harvester — синхронизация, расписание и мониторинг достаются бесплатно.
Свои инструменты AI
Каталог инструментов открытый: следующая итерация добавит подключение MCP-серверов с инструментами для агентов.
