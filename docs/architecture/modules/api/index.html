Admin Panel and Web App are facades: they hold no domain resources of their own, bringing owners' contracts together under a role. The Admin row is about the /admin/* config surface, not ownership.

Common rules
  • Version in the path — /api/v1 · /public/v1; breaking → a new version
  • Identity — a JWT or an ach_… key → user_id → role → ACL
  • Error — a code, not 500 — 401 · 403 · 404 · 409 · 422 · 429, one shape
  • Lists — one contract for pagination/filter/sort (list-controls)
  • Chat — an SSE stream: tokens incrementally, citations in the tail
  • Live boards — the server nudges, the client refetches over REST (Live updates)
  • Idempotency — queued by job_id: a retry doesn't double
  • Rate limit — window → 429 (Cache & Workers)
  • Time — UTC ISO 8601 (…Z); the timezone is applied by the frontend
Workzone Design

The layer's own mechanics: the platform push channel that keeps live boards fresh, and the conformance suite enforcing the rules above — error shape, UTC, pagination and rate limit checked parametrically across all routers, not by a copy in each module.
