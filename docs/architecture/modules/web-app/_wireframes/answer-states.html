Answer states

web app · wireframes shell

The same thread, the same message frame — but not every answer is grounded. The model answers in its own words, and our layer stitches a short badge under its answer: whether the knowledge base was searched and how it ended — empty, no access, or partly hidden. The badge carries safe coordinates: the source system and who to ask for access (the record's author, as an email). No red — the user got an answer; and for a conversational request Achilles replies like an ordinary assistant, with no badge at all. Four states — each with its own address.

The shell is the same everywhere — the thread and the message frame from assistant-ui (chat shell); what changes is our layer — the content of the assistant's message.
